# Лабораторна робота №1
## Дисципліна: Операційні системи
## Тема: “Знайомство з робочим середовищем віртуальних машин та особливостями операційної системи Linux”

**Мета роботи:**  
<span>1. Знайомство з гіпервізорами різного типу, віртуалізацією при роботі з операційними системами.</span>  
<span>2. Знайомство з основними видами сучасних ОС, короткий огляд їх можливостей.</span>  

**Матеріальне забезпечення занять:**
<span>1. ЕОМ типу IBM PC.</span>    
<span>2. ОС сімейства Windows та віртуальна машина Virtual Box (Oracle).</span>    
<span>3. ОС GNU/Linux (будь-який дистрибутив).</span>    
<span>4. Сайт мережевої академії Cisco netacad.com та його онлайн курси по Linux</span>  

**Завдання для попередньої підготовки.**

<span>1. Прочитайте короткі теоретичні відомості до лабораторної роботи та зробіть невеликий словник базових англійських термінів з питань класифікації віртуальних середовищ.</span>  

**Словник базових англійських термінів**

_1) **Virtual Machine (VM)** — віртуальна машина, програмна емуляція фізичного комп'ютера.  
2) **Hypervisor (Virtual Machine Monitor)** — гіпервізор, програмне забезпечення для створення та керування віртуальними машинами.  
3) **Type 1 Hypervisor (Bare Metal)** — гіпервізор 1-го типу, працює безпосередньо на апаратному забезпеченні.  
4) **Type 2 Hypervisor (Hosted)** — гіпервізор 2-го типу, працює як додаток усередині хостової ОС.  
5) **Host Operating System** — хостова (основна) ОС, операційне середовище, у якому запущено гіпервізор.  
6) **Guest Operating System** — гостьова ОС, операційна система, встановлена всередині віртуальної машини.  
7) **Partition** — розділ (диска), логічна частина фізичного диска, яку гіпервізори 1-го типу використовують для зберігання даних.  
8) **Emulate** — емулювати, відтворення роботи апаратних інструкцій за допомогою програмного забезпечення.  
9) **Interpreter** — інтерпретатор, програма, що виконує код (наприклад, JVM) крок за кроком.  
10) **Performance** — продуктивність/швидкість роботи, критичний показник при порівнянні різних типів віртуалізації.  
11) **Distinction** — розмежування/відмінність, ключова різниця між архітектурами гіпервізорів.  
12) **Booted** — завантажений, процес запуску операційної системи (як гостьової, так і хостової).  
13) **Interchangeably** — взаємозамінно, термін вказує на те, що назви "гіпервізор" та "монітор віртуальних машин" часто вживаються як синоніми.  
14) **Shipped over** — переданий/доставлений, у контексті Java вказує на можливість передачі байт-коду через мережу для виконання на будь-якому JVM-інтерпретаторі._

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2.1. Охарактеризуйте поняття «гіпервізор». Які бувають їх типи?
  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**Гіпервізор** — це програмний шар, який дозволяє одному фізичному комп'ютеру запускати декілька віртуальних машин. Кожна з них виглядає як окремий комп'ютер зі своєю ОС.   
  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;_Розрізняють наступні типи:_    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- Тип 1 (Bare metal). Не має під собою сторонньої ОС, керує залізом напряму (Xen, KVM).   
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- Тип 2 (Hosted). Працює "поверх" звичайної ОС (Windows чи Linux), використовуючи її файлову систему для зберігання віртуальних дисків (VirtualBox, VMware).
