| № | Слово | Пояснення |

| 1 | **Virtual Machine (VM)** | Віртуальна машина, програмна емуляція фізичного комп'ютера |  
| 2 | **Hypervisor (Virtual Machine Monitor)** | Гіпервізор, програмне забезпечення для створення та керування віртуальними машинами |
| 3 | **Type 1 Hypervisor (Bare Metal)** | Гіпервізор 1-го типу, працює безпосередньо на апаратному забезпеченні | 
| 4 | **Type 2 Hypervisor (Hosted)** | Гіпервізор 2-го типу, працює як додаток усередині хостової ОС |
| 5 | **Host Operating System** | Хостова (основна) ОС, операційне середовище, у якому запущено гіпервізор |

| 7 | **Partition** | Розділ (диска), логічна частина фізичного диска, яку гіпервізори 1-го типу використовують для зберігання даних |  
| 8 | **Emulate** | Емулювати, відтворення роботи апаратних інструкцій за допомогою програмного забезпечення |  
| 9 | **Interpreter** | Інтерпретатор, програма, що виконує код (наприклад, JVM) крок за кроком |  
| 10 | **Performance** | Продуктивність/швидкість роботи, критичний показник при порівнянні різних типів віртуалізації | 
| 11 | **Distinction** | Розмежування/відмінність, ключова різниця між архітектурами гіпервізорів |
| 12 | **Booted** | Завантажений, процес запуску операційної системи (як гостьової, так і хостової) |
| 13 | **Interchangeably** | Взаємозамінно, термін вказує на те, що назви "гіпервізор" та "монітор віртуальних машин" часто вживаються як синоніми |
| 14 | **Shipped over** | Переданий/доставлений, у контексті Java вказує на можливість передачі байт-коду через мережу для виконання на будь-якому JVM-інтерпретаторі |

**2.1. Охарактеризуйте поняття «гіпервізор». Які бувають їх типи?<br>**

**Гіпервізор** — це програмний шар, який дозволяє одному фізичному комп'ютеру запускати декілька віртуальних машин. Кожна з них 
виглядає як окремий комп'ютер зі своєю ОС.   
  
_Розрізняють наступні типи:_    
- Тип 1 (Bare metal). Не має під собою сторонньої ОС, керує залізом напряму (Xen, KVM).   
- Тип 2 (Hosted). Працює "поверх" звичайної ОС (Windows чи Linux), використовуючи її файлову систему для зберігання віртуальних 
дисків (VirtualBox, VMware).<br>  
 
**2.2. Перерахуйте основні компоненти та можливості гіпервізорів відповідно до свого варіанту (порядковий номер по журналу),

_Таблиця 1_  

| Варіант | 21 |

| Гіпервізор | VirtualBox |

**VirtualBox** - гіпервізор другого типу. Його основними компонентами та можливостями є використання хостової операційної системи та 
її файлової системи для створення процесів і зберігання віртуальних дисків у вигляді великих файлів. VirtualBox реалізує гібридну 
стратегію, що поєднує програмну емуляцію з модулем ядра для виконання складних операцій, що значно покращує продуктивність 
системи. Програма дозволяє запускати декілька гостьових ОС одночасно, забезпечуючи їхню повну ізоляцію, завдяки чому крах однієї 
віртуальної машини не впливає на роботу інших. Гіпервізор забезпечує таку поведінку гостьової ОС, що вона працює ідентично до 
функціонування на реальному апаратному забезпеченні, підтримуючи роботу графічних інтерфейсів та фонових процесів.</blockquote>

_Процес розгортання складається з таких основних кроків:<br>_

1\) завантаження та встановлення гіпервізора VirtualBox.<br>

**2.2. Чи є якісь апаратні обмеження при встановленні 32- та 64-бітних ОС?<br>**  

Так, для встановлення 64-бітних ОС процесор фізичного комп'ютера повинен підтримувати 64-бітну архітектуру та технології апаратної віртуалізації (Intel VT-x або AMD-V), які мають бути активовані в BIOS/UEFI. 
Якщо процесор 32-бітний, він не зможе запустити 64-бітну гостьову ОС. Також 32-бітні системи мають обмеження на використання оперативної пам'яті (зазвичай до 4 ГБ без спеціальних розширень).<br>  

**2.3. Які основні етапи при встановленні OS Linux в текстовому режимі?<br>**  

До встановлення в текстовому режимі зазвичай входить завантаження з інсталяційного носія та вибір пункту "Install" (Text mode), вибір мови, регіону та розкладки клавіатури. Також потрібно налаштувати мережеве ім'я (hostname) та облікові записи користувачів, розмітку дискового простору (створення розділів Root, Swap, Home). Необхідно обрати базові набори системного ПЗ та встановити завантажувач (наприклад, GRUB) на жорсткий диск.<br>  

**2.4. Яким чином можна до установити графічні оболонки Gnome та KDE в Linux, якщо вона вже встановлена в текстовому режимі (вкажіть необхідні команди та пакети)?<br>**   

Для встановлення графічних оболонок необхідно оновити список пакетів та скористатися пакетним менеджером (наприклад, apt для Debian/Ubuntu). 
- Для GNOME: _sudo apt update && sudo apt install ubuntu-desktop_ (або gnome-shell).
- Для KDE: _sudo apt update && sudo apt install kde-full_ (або kubuntu-desktop).
Після встановлення потрібно перезавантажити систему або запустити менеджер входу (gdm або sddm) командою _sudo systemctl start [назва_сервісу]_.<br>

### Контрольні запитання 
<div align="justify"> 
  
**<span>1. Порівняйте гіпервізори типу 1 та типу 2, яка між ними відмінність та сфера їх застосування?</span>**

- _Гіпервізори типу 1 (bare metal) працюють безпосередньо на апаратному забезпеченні та самостійно керують ресурсами, що забезпечує високу продуктивність. Вони зазвичай використовуються в серверних рішеннях та центрах обробки даних._
- _Гіпервізори типу 2 (hosted) працюють всередині хостової операційної системи та використовують її файлову систему для зберігання віртуальних дисків, що робить їх ідеальними для персонального використання та тестування ПЗ на домашніх ПК._

**<span>2. Розкрийте поняття «GNU GPL», яка його основна концепція?</span>**

_Основна концепція GNU GPL (General Public License) базується на філософії вільного програмного забезпечення, яка гарантує користувачам право отримувати вихідний код, вивчати його, змінювати та поширювати копії продукту. Це дозволяє спільноті програмістів вільно співпрацювати, покращуючи систему та забезпечуючи її відкритість для всіх._

**<span>3. В чому суть програмного забезпечення з відкритим кодом?</span>**

_Суть відкритого коду (Open Source) полягає в тому, що користувач отримує не просто готову програму, а її вихідні інструкції, які він має право модифікувати для власних потреб. Такий підхід значно прискорює темпи розробки та допомагає уникати помилок завдяки колективному аудиту коду багатьма учасниками._

**<span>4. Що таке дистрибутив?</span>**

_**Дистрибутив** — це набір програмного забезпечення, що включає ядро системи (Linux), набір інструментів (GNU), користувацькі додатки та пакетний менеджер, зібрані в єдиний інсталяційний пакет для зручного встановлення та керування системою._

**<span>5. Які задачі системного адміністрування можна реалізувати на базі ОС Linux?</span>**

_Linux дозволяє реалізовувати широкий спектр задач: від керування веб-серверами та базами даних до налаштування фаєрволів, моніторингу мереж та автоматизації рутинних процесів за допомогою скриптів. Більшість таких завдань професійно виконується через термінал за допомогою текстових команд._

**<span>6. Як пов'язані між собою ОС Android та Linux?</span>**

_ОС Android побудована на базі ядра Linux, яке відповідає за взаємодію з апаратним забезпеченням, керування пам'яттю та процесами. Linux виступає надійним фундаментом, на який Android додає власні графічні оболонки та специфічні програмні інтерфейси (API)._

**<span>7. Основні можливості та сфера використання Embedded Linux?</span>**

_Embedded Linux (вбудований Linux) використовується у пристроях спеціального призначення, таких як роутери, розумні телевізори, медичне обладнання та автомобільні системи. Його перевагами є можливість створення максимально легкої та оптимізованої системи під конкретне залізо при збереженні високої стабільності._

**<span>8. Яким чином можна змінити тип завантаження Linux: в текстовому режимі (3 рівень) або графічному (рівень 5)?</span>**

_Це можна реалізувати через налаштування системного завантажувача або зміну цільового режиму в системі ініціалізації. Сучасні дистрибутиви використовують команду systemctl set-default, де multi-user.target відповідає текстовому режиму, а graphical.target — графічному._

**<span>9. Чим відрізняються режими CLI та GUI?</span>**

_**GUI (графічний інтерфейс)** дозволяє взаємодіяти з системою через вікна та іконки за допомогою миші, що зручно для звичайних користувачів. **CLI (інтерфейс командного рядка)** — це текстовий метод керування через клавіатуру, який надає адміністратору набагато більше швидкості, точності та можливостей для автоматизації завдань._
