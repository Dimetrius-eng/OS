#### 2. Опишіть набір базових дій в встановленому Вами гіпервізорі:

- Створення нової віртуальної машини;

<blockquote>
  
Процес ініціалізації нової системи складається з наступних кроків:

**- Запуск майстра**: Натискання кнопки «Створити» (New) на головній панелі.  
**- Ідентифікація**: Вказання імені машини, вибір типу ОС (Linux) та версії (Ubuntu 64-bit).  
**- Вибір образу**: Підключення ISO-файлу операційної системи.

</blockquote>

- Вибір/додавання доступного для віртуальної машини обладнання;

<blockquote>
  
Гіпервізор дозволяє гнучко емулювати залізо у вкладці «Settings» (Налаштування):

**- System -> Motherboard:** Налаштування об’єму оперативної пам’яті (RAM). Для Ubuntu оптимально від 2048 MB.  
**- System -> Processor:** Вибір кількості віртуальних ядер (vCPU).  
**- Display:** Регулювання відеопам'яті (Video Memory). Для стабільної роботи графічних оболонок (GNOME/XFCE).  
**- Storage:** Додавання віртуальних жорстких дисків (VDI/VHD) або нових контролерів (SATA/NVMe).

<img src = "23.png" width = "800">

</blockquote>

- Налаштування мережі та підключення до точок Wi-Fi;

<blockquote>

У віртуальних машинах робота з мережею має свою специфіку. VM зазвичай не бачить Wi-Fi адаптер безпосередньо, а використовує «міст» через ПК:

**- Тип підключення NAT:** Віртуальна машина отримує інтернет через комп'ютер, але знаходиться в окремій внутрішній мережі.  
**- Тип підключення Bridged Adapter (Мережевий міст):** Машина стає повноправним членом домашньої мережі (отримує власну IP-адресу від роутера).  
**- Wi-Fi:** Щоб машина використовувала Wi-Fi, у налаштуваннях Network треба вибрати свій фізичний Wi-Fi адаптер у режимі "Bridged", тоді Linux сприйматиме його як дротове підключення зі швидкістю Wi-Fi.

<img src = "24.png" width = "800">

</blockquote>

- Можливість роботи з зовнішніми носіями (flash-пам’ять).
  
<blockquote>

Для того, щоб Ubuntu побачила реальну флешку, виконуються такі дії:

**- Встановлення Extension Pack:** На основну систему (Windows) потрібно встановити пакет розширень VirtualBox з [офіційного сайту](https://www.virtualbox.org/wiki/Downloads). Запускаємо завантажений файл — він автоматично відкриється у VirtualBox.

_Порівняння оболонок:_

| Критерій | GNOME (стандартна) | XFCE (додаткова) |
| :--- | :--- | :--- |
| Візуальний стиль | Сучасний, мінімалістичний, без класичного меню «Пуск» | Традиційний, схожий на класичні ОС Windows/Unix |
| Продуктивність | Споживає значну частину RAM (від 1 ГБ і вище) | Дуже легка, стабільно працює навіть на 512 МБ RAM |
| Керування | Орієнтоване на пошук та гарячі клавіші («Activities») | Орієнтоване на панелі, ярлики та звичні меню |
| Налаштування | Обмежені, потребує розширень | Можна змінити все: від розташування панелей до вигляду кожної кнопки |

#### Словник англійських термінів

| № | Слово | Пояснення |
| :--- | :--- | :--- |
| 1 | Hypervisor | Гіпервізор — програма для створення та керування віртуальними машинами (наприклад, VirtualBox) |
| 2 | Host OS | Хостова ОС — твоя основна операційна система (Windows), на якій встановлено гіпервізор |
| 3 | Guest OS | Гостьова ОС — операційна система, що працює всередині віртуальної машини (Ubuntu) |
| 4 | Terminal / CLI | Термінал / Інтерфейс командного рядка — текстовий інтерфейс для керування системою за допомогою команд |
| 5 | Desktop Environment (DE) | Графічна оболонка — набір візуальних елементів (вікна, панелі), як-от GNOME або XFCE |
| 6 | Guest Additions | Гостьові доповнення — пакет драйверів для покращення взаємодії між гостьовою та хостовою ОС |
| 7 | Repository | Репозиторій — онлайн-сховище, з якого Linux завантажує та встановлює програми |
| 8 | Package Manager (APT) | Менеджер пакетів — утиліта для встановлення, оновлення та видалення програмного забезпечення в Ubuntu |
| 9 | Mount / Unmount | Монтувати / Розмонтувати — процес підключення або відключення віртуального диска чи флешки до системи |
| 10 | Shared Clipboard | Спільний буфер обміну — функція, що дозволяє копіювати текст між Windows та віртуальною машиною |
| 11 | Snapshots | Знімки системи — збереження поточного стану віртуальної машини для швидкого відкату в разі помилок |
| 12 | Root / Sudo | Суперкористувач — режим адміністратора, що дає повний доступ до налаштувань системи |
